- (USE CASE DIAGRAM)
use case diagram is a way to show how a system or computer program works form the user's prespective
ketika kita ingin pake aplikasi, bakal ada orang yg menjelaskan cara pakai aplikasi, nah itu sudah termasuk usecase (tapi tidak dalam bentuk diagram),
simple nya untuk menjelaskan si user ini bisa ngapain aja 
ini ditujukan untuk menunjukkan sistem / program komputer berjalan dari pandangan seorang user.

- (symbol - symbol di diagram Use Case ):

AKtor -> (si user) 
Usecase -> (fitur apapun, menggambarkan fungsi atau fitur") -> 
Association Relationship -> hanya untuk mengabungkan elemen 
include relationship -> menggambarkan event/kegiatan sebelum terjadi
extend relationship -> menggambarkan event setelah dilakukan setelah event sebelumnya
generalisasi relationship -> -

yang paling penting adalah penambahan fitur apa yang dibutuhkan oleh user 
1. include(harus ke akses usecase untuk dapat akses fitu) / harus melewati dulu
2. extend (untuk akses fitur dalam fitur) / perpanjang fitur
